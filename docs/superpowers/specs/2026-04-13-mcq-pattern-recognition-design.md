# MCQ Pattern Recognition — Design Spec

## Context

The study app teaches via 4 understanding layers (story/analogy/because/failure) and stores full exam answers (L5). For an ADHD+dyslexia brain, active recall practice with *near-miss* wrong answers trains the pattern recognition needed in an exam where options are subtly different, not obviously wrong.

## What

Add MCQ practice to each concept, integrated into the concept explorer view. One correct answer + three close wrong answers, auto-generated from the concept corpus. Wrong answers are *near-miss* distractors — same module, similar wording, plausible — forcing careful discrimination rather than elimination of the obviously wrong.

## Data Model

```json
{
  "concepts": [{
    "id": "Q3",
    "module": 1,
    "topic": "Turing Test",
    "layers": { "L1": "...", "L2": "...", "L3": "...", "L4": "...", "L5": "..." },
    "mcq": {
      "question": "What is the Turing Test?",
      "options": [
        "A test where a machine tries to convince a judge it's human in conversation",
        "A benchmark for measuring model inference speed under compute constraints",
        "A method for evaluating feature importance in supervised learning models",
        "A protocol for testing model robustness against adversarial inputs"
      ],
      "correct": 0,
      "autoGenerated": true
    }
  }]
}
```

- `options[0]` is always the correct answer (identified by index in `correct`)
- Wrong options sourced from the same module's concept topics (auto-generated)
- LocalStorage can store user-edited MCQs that override auto-generated ones

## MCQ Generation Algorithm (auto-generated)

For concept X in module M with topic T:

1. **Same-module candidates**: collect all other topics in module M
2. **Near-miss selection**: score each candidate by string similarity to T — prefer topics with shared structural words (e.g., "vs" pairs, "What is / What are" variants)
3. **Plausibility guarantee**: if no same-module candidate is close enough, pull from any module using shared keyword overlap
4. **Shuffle**: options array is shuffled so correct answer isn't always in position 0

Wrong answers are created by taking the *topic title* of the distractor concept and expanding it slightly — short enough to look like a plausible option, different enough to be wrong.

## UX Flow

**Concept Explorer:**
- Each concept card gets a new "MCQ Practice" button above the layer reveals
- Clicking opens an inline MCQ widget below the concept title
- Shows question + 4 radio options; user selects one, clicks "Check Answer"
- Correct: green highlight on selected, "Correct!" message, L1–L4 reveal is unlocked
- Wrong: red highlight on selected, correct answer pulses green, brief explanation shown
- After answering: L1–L4 layers unlock automatically
- MCQ state (answered/not) stored in LocalStorage — revisited concepts show "MCQ ✓" badge

**Quiz Mode:**
- Full quiz sessions remain unchanged (active recall via "Try to recall" reveal)
- MCQ is supplementary in the explorer, not the quiz

**Keyboard shortcut:** `M` — open MCQ for current concept (same concept as current card focus)

## LocalStorage Schema

```json
{
  "mcq_answers": {
    "Q3": { "answered": true, "correct": true, "attempted_at": 1713000000 }
  },
  "mcq_custom": {
    "Q3": { "options": [...], "correct": 0 }
  }
}
```

- `mcq_answers` — tracks per-concept MCQ history
- `mcq_custom` — user-edited overrides (merges with auto-generated at render time)

## Scope for Version 1

- MCQ button on concept card in explorer mode only
- Auto-generated options from same-module topic titles
- Correct answer always in options (no fabrications)
- No quiz-mode MCQ integration (out of scope for v1)
- No MCQ editing/authoring UI in v1 (auto-generated only)
- No LLM/API calls — fully offline

## Out of Scope (Future)

- User-authored MCQs
- MCQ sessions as a separate study mode
- Spaced retrieval based on MCQ wrong answers
- Per-MCQ confidence tracking separate from layer confidence
